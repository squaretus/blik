# blik

Мониторинг системы и управление вентиляторами для Mac на Apple Silicon.

https://blik-app.ru

Blik показывает в реальном времени температуры, загрузку CPU, GPU, памяти и диска, а также обороты кулеров, ведёт локальную историю метрик с графиками и позволяет вручную управлять скоростью встроенных вентиляторов. Данные читаются напрямую через SMC (System Management Controller); привилегированный XPC-демон выполняет операции с root-правами, поэтому всё работает без `sudo`.

## Возможности

- **Обзор** — ключевые метрики системы одним экраном: температуры и загрузка CPU / GPU / RAM / VRAM / диска
- **Температура** — все датчики (CPU, E-Core, GPU, SSD, батарея и др.) с цветовым кодированием
- **Ресурсы** — загрузка CPU (P- и E-ядра), GPU, использование RAM и VRAM, скорость чтения/записи диска
- **Графики** — локальная история метрик за период до 7 дней (live-режим и исторические периоды)
- Управление скоростью кулеров через пресеты: 0% (авто), 25%, 50%, 75%, 100%
- Инлайн-переименование датчиков и метрик
- Автоматический возврат кулеров в авто-режим при выходе и защита от «осиротевшего» состояния
- Три интерфейса: приложение в строке меню, приложение с дашбордом, CLI-утилита
- **Интеграция с Claude Code**: метрики в статус-баре и MCP-сервер (Claude может сам смотреть температуры и управлять кулерами)
- Автообновление через GitHub Releases

## Установка

1. Скачайте `Blik-x.x.x.pkg` из [Releases](../../releases/latest).
2. Откройте файл. macOS заблокирует запуск — PKG не подписан Developer ID, это ожидаемо.
3. Разрешите запуск одним из способов:
   - **Системные настройки → Конфиденциальность и безопасность** → внизу появится сообщение о заблокированном файле → **«Всё равно открыть»**.
   - Либо в терминале: `xattr -d com.apple.quarantine ~/Downloads/Blik-*.pkg`, затем откройте PKG повторно.
4. Пройдите визард и введите пароль администратора.

После установки демон запущен, приложение в строке меню стартует при входе в систему (можно отключить в **Настройки → Основные → Объекты входа**), а команда `blik` доступна в терминале.

## CLI

```bash
blik                     # полноэкранный терминальный дашборд (TUI)
blik --once              # однократный вывод метрик и выход
blik --read-only         # только мониторинг, без управления кулерами
blik --update            # проверить и установить обновление
```

В TUI: клавиши `1`–`5` — пресеты кулеров (`1` = авто), `↑/↓` — скролл сенсоров, `Q` — выход с возвратом авто-режима.

## Интеграция с Claude Code

Требуется установленный blik (см. «Установка») и [Claude Code](https://claude.com/claude-code).

### Быстрая установка (плагин)

Репозиторий blik — это ещё и маркетплейс плагинов Claude Code. В Claude Code выполните:

```
/plugin marketplace add squaretus/blik
/plugin install blik@blik
```

Плагин подключает MCP-сервер (см. ниже) и добавляет команду `/blik:setup-statusline`, которая настраивает вывод метрик в статус-бар — в том числе аккуратно дописывает их в уже существующий statusline-скрипт, если он у вас есть.

Разделы ниже — то же самое вручную, без плагина.

### Метрики в статус-баре

Команда `blik claude-statusline` печатает одну строку: температуры CPU (P-ядра), E-ядер и GPU, занятая RAM и VRAM — каждая метрика с графиком-спарклайном за последние 30 минут. Цвет значения отражает уровень: зелёный — норма, жёлтый — от 70 °C (память — от 70% заполнения), красный — от 90 °C (90%).

Минимальная настройка — добавьте в `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "/usr/local/bin/blik claude-statusline",
    "refreshInterval": 5
  }
}
```

Если у вас уже есть свой statusline-скрипт — просто вызовите `blik claude-statusline` в его конце: вывод метрик станет последней строкой статус-бара.

Если демон blik недоступен, команда молча выводит пустую строку — статус-бар не ломается.

### MCP-сервер

Команда `blik mcp` поднимает MCP-сервер (stdio), через который Claude сам запрашивает метрики и управляет кулерами. Регистрация:

```bash
claude mcp add --scope user blik -- /usr/local/bin/blik mcp
```

Инструменты:

| Инструмент | Что делает |
|---|---|
| `get_current_metrics` | Текущие температуры (все сенсоры и средние), загрузка CPU/GPU, RAM/VRAM, обороты кулеров |
| `list_metrics` | Ключи метрик, доступных в локальной истории |
| `query_metric_history` | История метрики (min/avg/max) за период до 7 дней |
| `set_fan_preset` | Пресет скорости кулеров: 0 (авто) / 25 / 50 / 75 / 100% |

Примеры запросов к Claude: «какая сейчас температура GPU?», «покажи историю RAM за час», «поставь кулеры на 50%», «верни кулеры в авто».

`set_fan_preset` управляет физическими вентиляторами — Claude Code спросит подтверждение перед вызовом, как для любого MCP-инструмента.

## Удаление

Полное удаление (приложение, демон, CLI, логи, кэши, разрешения) — любым из способов:

- **Из приложения:** строка меню → ⚙ → **«Удалить Blik»**.
- **Через Finder:** перетащите `Blik.app` из `/Applications` в корзину — демон обнаружит удаление и очистит остальные компоненты.
- **Через терминал:**
  ```bash
  cp /Applications/Blik.app/Contents/Resources/uninstall-helper.sh /tmp/
  bash /tmp/uninstall-helper.sh
  ```

## Обновление

Демон проверяет наличие новых версий через GitHub Releases при запуске и каждые 6 часов.

- **Строка меню:** баннер «Доступно обновление» с кнопкой «Обновить» или ⚙ → «Проверить обновления».
- **CLI:** `blik --update`.

Обновление выполняется демоном с root-правами: скачивание PKG → тихая установка → перезапуск компонентов.
